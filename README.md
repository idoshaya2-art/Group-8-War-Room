# Group 8 — EMBA 41 — TAU · INTOPIA War Room

פלטפורמת תמיכה בהחלטות לסימולציית INTOPIA (חברה 8). אפליקציית דפדפן חד-קובצית,
עברית RTL מלאה, ללא התקנה, עם שמירה אוטומטית מקומית (localStorage).

## הפעלה
פתחו את `index.html` בדפדפן, או דרך GitHub Pages. דורש חיבור אינטרנט לטעינת
ספריית קריאת האקסל (SheetJS) והפונטים.

## מודולים (MVP)
- **קליטת דוח (F1)** — גרירת קובץ XLSX, זיהוי גיליונות (Balance/Income/MR*), מסך אימות ותיקון.
- **דשבורד ונורות (F3)** — מדדי-על, 6 נורות אזהרה אוטומטיות (כולל גירעון מטבע), מגמות ותדריך פתיחה.
- **מנוע תזרים (F4)** — הקרנת מזומן דטרמיניסטית עד Q9 מול רצפות, פירוק לפי אזור ומטבע.
- **מודיעין (F5)** — מאגר MR מצטבר וממליץ מחקרים.
- **AI / מועצה (F7)** — צ׳אט מעוגן-נתונים עם ציטוט מקור + מצב מועצה (5 פרספקטיבות, סינתזה). דורש מפתח Anthropic API אישי (נשמר מקומית בלבד, לא בגיבוי).
- **סימולטור (F6)** — מנופים לכל רבעון, שמירת תרחישים, השוואה וניתוח רגישות ±10%.
- **ייצוא הזנה (F8)** — גיליון בפורמט טפסי INTOPIA (H/A/W) + צ׳ק-ליסט לפני Submit + יומן החלטות.
- **יעדים (F2)** — רצפות, חוזים ופרמטרי מודל (גבייה/ריבית/מס/שערי מטבע) — האסטרטגיה היא קונפיגורציה.
- **MAX — מיקסום Q9** — אופטימיזציה: המנוע סורק שילובי מנופים (עצימות ייצור, שולי מחיר, מו״פ) ובוחר את התוכנית שממקסמת את פרוקסי ציון הסיום ב-Q9 מבלי לשבור רצפות. התוצאה נשמרת כתרחיש בסימולטור.

## שמירת נתונים וסנכרון צוותי (F9)
- **שמירה מקומית** אוטומטית (localStorage) לאחר כל שינוי.
- **גיבוי / שחזור** — ייצוא/ייבוא קובץ JSON, גם כמסלול שיתוף ידני.
- **סנכרון צוותי חי** (כפתור ☁️ סנכרון) — מצב משותף לכל הצוות בזמן אמת דרך
  Firebase Realtime Database (bring-your-own, חינם). כל הצוות מזין אותו URL + קוד
  חדר; שינוי אצל אחד מתעדכן אצל כולם, עם סימון "מי ערך אחרון". מודל last-write-wins,
  מתאים לעבודה כיתתית תורית. הגדרות הסנכרון נשמרות מקומית בלבד ואינן בגיבוי.

עיקרון-על: עובדות, מודל ויעדים מופרדים לשלוש שכבות. כל המספרים מחושבים במנוע דטרמיניסטי.

> גרסה קודמת (אפליקציית וויסות רכבים) נשמרה בקובץ `vehicle-app.html`.
